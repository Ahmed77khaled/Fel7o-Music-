# Fel7o Downloader

برنامج ويندوز لتحميل الصوت والفيديو من يوتيوب — فيديو منفرد أو قائمة تشغيل كاملة، بجودة قابلة للاختيار، مع سجل تحميلات وإدارة قوائم انتظار.

> **الإصدار الحالي:** 5.0.1

## 👤 للمستخدم

### المتطلبات

- ويندوز 10/11 (64-bit)
- اتصال بالإنترنت

### التثبيت

1. حمّل ملف `Fel7o Downloader Setup.exe` من صفحة الإصدارات (Releases).
2. شغّل الملف واتبع خطوات التثبيت (تقدر تختار مكان التثبيت).
3. بعد التثبيت هتلاقي اختصار على سطح المكتب وفي قائمة ابدأ.

> **ملاحظة:** البرنامج بيجيلك جاهز بكل احتياجاته (yt-dlp وffmpeg مدمجين جوه). مفيش أي تثبيت إضافي مطلوب منك.

### طريقة الاستخدام

1. **انسخ رابط** فيديو أو قائمة تشغيل من يوتيوب.
2. الصقه في خانة الرابط بالبرنامج (لو مفعّل "لصق تلقائي من الحافظة" في الإعدادات، هيتحط لوحده).
3. اختر:
   - **الوضع**: صوت (MP3) أو فيديو.
   - **الجودة**: حسب الخيارات المتاحة لكل وضع.
4. لو الرابط قائمة تشغيل، هيفتح **مدير قوائم التشغيل** يخليك تختار الفيديوهات اللي عايزها بالظبط، وبيوريك المدة الكلية والحجم التقريبي قبل ما تحمّل.
5. اضغط **تحميل** — التحميل هيتضاف لقائمة الانتظار وتقدر:
   - توقف/تكمل أي تحميل فرديًا.
   - توقف/تكمل كل التحميلات مرة واحدة.
   - تلغي تحميل في أي وقت.
6. بعد ما يخلص، تقدر تفتح مجلد الملف مباشرة من البرنامج.

### الإعدادات

- **مجلد التحميل**: تقدر تغيّره من الإعدادات.
- **عدد التحميلات المتزامنة**: كام تحميل يشتغلوا مع بعض في نفس الوقت.
- **جودة الصوت/الفيديو الافتراضية**.
- **تضمين الصورة المصغّرة والبيانات الوصفية (Metadata)** في الملف الناتج.
- **اللصق التلقائي من الحافظة**.

### السجل (History)

كل تحميل ناجح بيتسجل في السجل، وتقدر تدور فيه أو تمسحه بالكامل أو تمسح عنصر واحد.

### استكشاف الأخطاء

| المشكلة | الحل |
|---|---|
| رسالة "yt-dlp غير موجود" | أعد تثبيت البرنامج — الملف المفروض يكون موجود جوه مجلد التثبيت تلقائيًا |
| رسالة "ffmpeg غير موجود" | نفس الحل أعلاه |
| "تعذر تحميل بيانات القائمة" | تأكد إن قائمة التشغيل **عامة** (public) مش خاصة أو غير مدرجة |
| التحميل بيفشل برسالة غير واضحة | افتح "السجل" أو راقب رسالة الخطأ التفصيلية اللي بتظهر تحت شريط التحميل |

## 🛠 للمطور

### البنية العامة (Architecture)

المشروع تطبيق **Electron** بمعمارية Main/Renderer قياسية مع `contextBridge` للعزل الآمن:

```
├── main.js          # Main process: نوافذ، IPC handlers، تشغيل yt-dlp/ffmpeg
├── preload.js        # جسر آمن (contextBridge) بين main والـ renderer
├── src/
│   ├── index.html     # الواجهة
│   ├── renderer.js    # منطق الواجهة (state, rendering, event handlers)
│   └── styles.css     # التصميم
├── bin/               # yt-dlp.exe + ffmpeg.exe (يتحطوا هنا بعد postinstall)
├── scripts/
│   └── extract-ffmpeg.js  # بيفك ffmpeg.zip تلقائيًا بعد npm install
└── package.json
```

**قواعد أمان أساسية متبعة:**
- `contextIsolation: true`, `nodeIntegration: false`, `sandbox: true`.
- كل التواصل بين الواجهة والـ main process بيعدي حصريًا عن طريق IPC channels معرّفة في `preload.js` (مفيش أي `require` مباشر من الـ renderer).
- كل استدعاءات `spawn`/`execFile` بتستخدم args array (مفيش `shell: true`) — يمنع command injection حتى مع مدخلات غير موثوقة.
- أي رابط بيتبعت لـ yt-dlp بيتفحص أولاً إنه رابط يوتيوب صحيح (`isValidYoutubeUrl` في `main.js`).

### تشغيل المشروع محليًا

```bash
git clone https://github.com/Ahmed77khaled/Fel7o-Music.git
cd Fel7o-Music
npm install     # postinstall بيفك ffmpeg.zip تلقائيًا
npm start
```

### بناء نسخة exe للتوزيع

```bash
npm run dist
```

الناتج هيبقى في `release/win-unpacked/` (نسخة غير مضغوطة) وملف installer باسم مشابه لـ `Fel7o Downloader Setup <version>.exe` في `release/`.

> قبل البناء النهائي، تأكد إن `assets/icon.ico` موجود — ده اللي بيظهر كأيقونة للبرنامج والـ installer (معرّف في `package.json > build.win.icon`).

### خريطة الـ IPC (Main ↔ Renderer)

| القناة | الاتجاه | الوظيفة |
|---|---|---|
| `settings:get` / `settings:save` | invoke | قراءة/حفظ الإعدادات |
| `history:get` / `history:add` / `history:clear` / `history:delete` | invoke | إدارة سجل التحميلات |
| `dialog:chooseFolder` | invoke | اختيار مجلد تحميل |
| `shell:openFolder` / `shell:openJobFolder` / `shell:openExternal` | invoke | فتح مجلدات/روابط خارجية |
| `tools:checkFfmpeg` / `tools:checkYtdlp` | invoke | التأكد من توفر الأدوات |
| `ytdlp:getInfo` / `ytdlp:getPlaylistInfo` | invoke | جلب بيانات فيديو/قائمة تشغيل |
| `download:start` / `pause` / `resume` / `pauseAll` / `resumeAll` / `cancel` | invoke | التحكم في التحميلات |
| `download:progress` / `done` / `error` / `cancelled` / `log` | on (من main لـ renderer) | تحديثات حالة التحميل الفورية |

### ملاحظات تصميم مهمة

- كل عمليات التحميل بتتعامل معاها عن طريق فنكشن مشتركة `attachProcessHandlers()` في `main.js` — أي تعديل على منطق قراءة الـ output أو الأخطاء لازم يتعمل هنا بس، مش في 3 أماكن منفصلة.
- تقدير حجم الملف في مدير قوائم التشغيل (`estimateVideoBytes` في `renderer.js`) هو **تقدير حسابي** بناءً على الجودة المختارة والمدة، مش حجم حقيقي مجلوب من yt-dlp، وده مقصود (شرح السبب موجود في تعليق الكود).

### المساهمة

لو حابب تضيف feature أو تصلح باگ:
1. افتح issue بوصف المشكلة/الاقتراح.
2. تأكد إن أي تعديل في `main.js` بيحافظ على استخدام `attachProcessHandlers` بدل تكرار كود الـ process handlers.
3. شغّل `node -c main.js` (وبالمثل لباقي ملفات JS) قبل الـ commit للتأكد من خلو الكود من أخطاء syntax.
